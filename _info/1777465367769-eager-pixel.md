## Executive Summary

The Daily Property Show is an automated system that discovers, scores, and selects 6 European real estate properties daily from 13 boutique estate agents. The system uses a "Heart-Rate" algorithm that evaluates properties based on **emotional impact** rather than just specs:

- **Sublime Escapism** (35%) - Views, privacy, land size
- **Authentic Bones** (30%) - Historic character features
- **Sanctuary Capacity** (20%) - Space to build dreams (shala, sauna, studio)
- **Manageable Project Multiplier** - Safety check on price/condition

Output: 6 diverse properties delivered via Telegram each morning with production-ready CSV sheets for YouTube filming.

**Cost:** $0/month (all free tiers)  
**Tech Stack:** Python + Playwright + SQLite + GitHub Actions + Cloudflare D1 + FastAPI  
**Agencies:** 13 across France, Italy, Spain, Portugal

## Heart-Rate Algorithm

### The 3 Pillars

#### 1. Sublime Escapism (35% weight)
Measures the "wow factor" of location and setting.

```
Score components:
- Sea view: +40 pts
- Mountain view: +35 pts
- Valley/panoramic view: +30 pts
- Neighbor distance >100m: +20 pts
- Land size (hectares × 5): up to +25 pts
```

**Why it matters:** You can renovate a kitchen, but you can't install a sea view. This is the heaviest weight because location is everything.

#### 2. Authentic Bones (30% weight)
Measures historic character and soul.

```
Score components:
- Exposed beams: +25 pts
- Original stone/terracotta floors: +25 pts
- Functional fireplaces: +15 each (max 45)
- Structural stone walls: +20 pts
- Wooden ceilings: +15 pts
- Construction year <1900: +20 pts
```

**Why it matters:** Authentic features are irreplaceable. They signal the property has genuine character that modern builds can't replicate.

#### 3. Sanctuary Capacity (20% weight)
Measures space to build your dream lifestyle.

```
Score components:
- Outbuildings: 1×20, 2×40, 3+×60 pts
- Annex potential: +25 pts
- Flat land >1000m²: +20 pts
- Pool: +15 pts
- Separate entrance: +15 pts
- Set back from road: +10 pts
```

**Why it matters:** Engages imagination. Can you put a yoga shala in that barn? A sauna in that separate annex? This score captures the "potential" factor.

### Manageable Project Multiplier (Safety Check)

Multiplier ranges 0.1-1.0. Applied to raw score to filter out unmanageable properties:

```
Affordability:
  price > €250k → ×0.3 (too expensive)
  price < €30k → ×0.7 (suspiciously cheap)
  price < €80k → ×0.9 (budget-friendly bonus)

Condition:
  shell/ruin → ×0.2 (major rebuild)
  renovation_needed → ×0.7 (moderate work OK)
  habitable → ×1.0 (ready to live)

Size sanity:
  bedrooms > 8 → ×0.5 (mansion penalty)
  living_area > 500m² → ×0.7 (high upkeep)
```

### Final Score Calculation

```
raw = (sublime × 0.35) + (authentic × 0.30) + (sanctuary × 0.20)
novelty_bonus = unique_keyword_matches × 15
final_score = (raw + novelty_bonus × 0.15) × multiplier
```

**Range:** 0-1000 points (display-friendly)

### Segment Assignment

Properties automatically assigned to one of 6 show segments:

1. **The Sublime View** - sublime score ≥ 75
2. **The Authentic Bones** - authentic score ≥ 75
3. **The Sanctuary Plot** - sanctuary score ≥ 70
4. **The Quick Win** - price < €80k, condition habitable/renovation, authentic ≥ 60
5. **The Unique Wonder** - has unique features (mill, tower, cave) OR novelty bonus > 30
6. **The Balanced Gem** - everything else (well-rounded)

## Daily Workflow

### 6:00 AM - Scrape Trigger (GitHub Actions)

Workflow `.github/workflows/daily-property-scrape.yml`:

1. Checkout code
2. Setup Python 3.11
3. Install dependencies + Playwright browsers
4. Run `python -m orchestrator`
5. Archive logs & database snapshot
6. Commit database changes back to repo
7. Generate production sheet artifact

**Duration:** ~30 minutes  
**Free tier usage:** ~900 minutes/month (well under 2000 limit)

### 7:00 AM - Selection Engine

Runs inside orchestrator after scraping:

1. Load all properties from database (last 3 days verified)
2. Filter eligible (has price, bedrooms, not sold)
3. Apply any curator preferences (price max, countries)
4. Remove properties shown in last 90 days
5. Score all with Heart-Rate algorithm
6. Enforce geographic diversity (max 2 per city/cluster)
7. Select top candidate for each of 6 segments
8. Find nearby past episodes (within 20km)
9. Output selection to database + Telegram

### 7:30 AM - Telegram Notification

Message format:

```
🏡 Daily Property Show - 2026-04-29
📊 Avg Score: 412/1000

1. The Authentic Bones
   Stone Farmhouse with Outbuildings in Saint-Cyprien, Dordogne
   💰 €185,000 | 🛏️ 4 bed | 📍 Saint-Cyprien, France
   🔗 https://listing-url...

[6 properties total]
```

Sent to curator's Telegram chat for quick review.

### 8:00 AM - Human Review (Dashboard)

Dashboard at `http://localhost:5000` (or GitHub Pages):

- View all 6 properties with full details
- See Heart-Rate breakdown (sublime/authentic/sanctuary)
- Primary image displayed
- Nearby past episodes indicated
- Backup pool of 10 alternates visible
- **Optional:** Swap any property with backup (persisted to DB)

If no action by 9 AM, selection auto-approves.

### 9:00 AM - Production Sheet Export

CSV format generated:

| Segment | Title | Price | Beds | City | Country | Heart Score | Sublime | Authentic | Sanctuary | URL | Google Maps | Talking Points |
|---------|-------|-------|------|------|---------|-------------|---------|-----------|-----------|-----|------------|----------------|

Saved to `output/production-sheet-YYYY-MM-DD.csv`

Also generates JSON version for programmatic access.

### 10:00 AM - Ready to Shoot

Production sheet downloaded, Google Earth placemarks created (manual step for now), filming begins.


## Troubleshooting

### Scraper Getting 403/429 (Blocked)

**Symptoms:** GitHub Actions logs show "HTTP 403" or "429 Too Many Requests"

**Solutions:**
1. Increase `scrape_delay` in scraper class (currently 2.0 seconds)
2. Add rotating user agents
3. Implement proxy rotation (BrightData, ScraperAPI)
4. Contact agent directly to request API access or whitelist

### Properties Not Being Selected

**Symptoms:** Selection returns fewer than 6 properties

**Check:**
1. Are properties scoring too low? Check average heart_rate_score in DB
2. Is deduplication too aggressive? Check how many properties in `showed_properties` table
3. Is geographic diversity eliminating too many? Check cluster distribution

**Quick fix:** Reduce deduplication window from 90 to 60 days in `PropertySelector._exclude_recent()`

### Telegram Not Sending

**Check:**
1. Bot token valid: `curl https://api.telegram.org/bot<token>/getMe`
2. Chat ID correct: Can be numeric or @channel
3. Bot has permission to post in chat (for groups)
4. No network issues in GitHub Actions

### Database Growing Too Fast

**Symptoms:** D1 storage approaching 5GB limit

**Solutions:**
1. Archive old shows: Move `shows` older than 1 year to separate table
2. Prune `price_history`: Keep only last 6 months
3. Compress `scrape_log` entries older than 90 days
4. Increase scraper frequency only as needed (currently daily)

### Score Seemingly Wrong

**Debug:** Use `demo_final.py` with sample property data to test algorithm in isolation.

Common issues:
- Description text not being parsed for keywords (check regex patterns)
- Price not extracted correctly (check `extract_price()` for locale formatting)
- Condition not detected (extend `CONDITION_KEYWORDS` in scorer)

## "Nearby" Feature Logic

### Goal: Keep viewers engaged by referencing earlier episodes

```sql
-- Find properties within radius of current property
SELECT p.*, 
  (SELECT show_id FROM showed_properties sp 
   WHERE sp.property_id = p.id 
   LIMIT 1) as episode_ref
FROM properties p
WHERE haversine(
  :current_lat, :current_lon,
  p.latitude, p.longitude
) < 20  -- 20km radius
AND p.id NOT IN (current_show_properties)
AND p.last_verified > NOW() - INTERVAL 30 DAY
ORDER BY p.date_listed DESC
LIMIT 3;
```

## Algorithm Scoring Formula (The "Heart-Rate" Engine)

```python
def calculate_heart_rate_score(property):
  # Sublime Escapism Index (0-100)
  sublime = (
    (has_sea_view ? 40 : 0) +
    (has_mountain_view ? 35 : 0) +
    (has_valley_view ? 30 : 0) +
    (neighbor_distance_m > 100 ? 20 : 0) +
    (land_area_ha * 5)  # Up to 25pts for >5ha
  )

  # Authentic Bones Score (0-100)
  authentic = (
    (has_exposed_beams ? 25 : 0) +
    (has_original_stone_floors ? 25 : 0) +
    (has_functional_fireplaces * 15) +
    (has_structural_stone_walls ? 20 : 0) +
    (has_wooden_ceilings ? 15 : 0) +
    (construction_year > 1800 ? 20 : 0)  # Historic bonus
  )

  # Sanctuary Capacity (0-100)
  sanctuary = (
    (outbuilding_count * 20) +  # Up to 60 for 3+ outbuildings
    (has_annex_potential ? 25 : 0) +
    (flat_land_area_m2 > 1000 ? 20 : 0) +
    (has_pool ? 15 : 0) +
    (has_separate_entrance ? 15 : 0) +
    (is_set_back_from_road ? 10 : 0)
  )

  # Manageable Project Multiplier (0.1 - 1.0)
  # BASELINE SAFETY CHECK - prevents overhyped disasters
  multiplier = 1.0

  # Affordability check
  if price_eur > 250000:
    multiplier *= 0.3  # Price ceiling penalty
  elif price_eur < 30000:
    multiplier *= 0.7  # Too cheap = suspicious
  elif price_eur < 80000:
    multiplier *= 0.9  # Budget-friendly bonus

  # Structural soundness
  if condition == "shell" or condition == "ruin":
    multiplier *= 0.2  # Requires major rebuild
  elif condition == "renovation_needed":
    multiplier *= 0.7  # Moderate work acceptable
  elif condition == "habitable":
    multiplier *= 1.0  # Ready to live

  # Size reasonableness
  if bedrooms > 8:
    multiplier *= 0.5  # Too large = high upkeep
  if living_area_m2 > 500:
    multiplier *= 0.7  # mansion penalty

  # Final score (weighted)
  raw_score = (
    sublime * 0.35 +      # Heaviest - view is unchangeable
    authentic * 0.30 +    # Second - character matters
    sanctuary * 0.20 +    # Third - potential for dreams
    novelty_bonus * 0.15  # Unique features
  ) * multiplier

  return raw_score
```

**Data Extraction Priority:** The scrapers must capture these specific emotional triggers from listing descriptions and photos:
- View mentions: "sea view", "panoramic", "vista", "overlooking"
- Authentic features: "original beams", "terracotta floors", "fireplace", "stone walls"
- Sanctuary potential: "outbuilding", "barn", "annex", "separate entrance", "gîte"
- Condition keywords: "habitable", "renovate", "shell", "mint condition"
